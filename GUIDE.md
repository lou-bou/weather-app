Node.js 18+

# Quick Start

```bash
npm install
npm run dev
```

### Install dependencies

`npm install`

### Project Structure

`src/` - application source code.

`src/index.js` - main JavaScript entry point.

`src/template.html` - HTML template used by HtmlWebpackPlugin.

`dist/` - build output generated by webpack. Do not edit manually and do not commit to Git.

All other files in the root directory are configuration files.

##### Webpack configuration files

`webpack.common.js` - a common configuration to keep things DRY.

`webpack.dev.js` - configuration for the dev environment.

`webpack.prod.js` - configuration for the prod environment.

##### Important files

Aside from the config stuff in root, these files shouldn't be renamed (unless you also update the path in `webpack.common.js`) nor deleted:

- `src/index.js`
- `src/template.html`

`src/styles.css` can be renamed or deleted freely.

### Dev Dependencies

`webpack` - the module bundler.

`webpack-cli` - command line interface for running webpack.

`webpack-dev-server` - development server with live reload.

`webpack-merge` - utility for combining multiple webpack configs.

`html-webpack-plugin` - generates the final HTML file and injects bundles.

`style-loader` - injects CSS into the DOM.

`css-loader` - resolves CSS imports and URLs.

`html-loader` - processes HTML and resolves asset references.

`eslint` - Linter.

`prettier` - Formatter.

### Static Assets

Images, fonts, and other assets can be placed anywhere inside `src/`.

Example:

`src/assets/logo.png`

`src/images/background.jpg`

### Usage

#### Exporting/Importing JS modules

**Exporting:**

```js
// src/external.js
export const myConst = 1234;
```

**Importing:**

```js
// src/index.js
import { myConst } from "./external.js";

console.log(myConst);
```

#### Importing CSS files

```js
// src/index.js
import "./styles.css";
```

#### Loading Images

**In CSS:**

```css
div {
  background-image: url("./star.gif");
}
```

**In HTML:**

```html
<img src="./star.gif" />
```

**In JS:**

```js
import starGif from "./star.gif";

const image = document.createElement("img");
image.src = starGif;

document.body.appendChild(image);
```

#### Deploying in Github Pages

1. Make a new branch: `git branch gh-pages`. _Only do this the first time you deploy._
2. Commit all work.
3. Change branch and sync changes from `main`: `git checkout gh-pages && git merge main --no-edit`.
4. Bundle the app: `npm run build`.
5. Run these commands in order:

```bash
git add dist -f && git commit -m "Deployment commit"
npm run deploy
git checkout main
```

6. Change source branch in Github Pages to `gh-pages`.

### Linting & Formatting

#### Linting

The linter used is **ESLint**.

To use it, run: `npm run lint`. This will lint all files in `src/`.

The current config uses the recommended ESLint linting rules. You can modify the rules in `eslint.config.js`.

#### Formatting

The formatter used is **Prettier**.

To use it through the CLI:

- To check for formatting issues: `npm run check-format`.
- To apply Prettier formatting: `npm run format`.

The current config uses the default Prettier formatting rules (`.prettierrc`).

You can choose which files not to format by adding them to `.prettierignore`.

Make sure to also set up Prettier in your editor. See [Editor Integration](https://prettier.io/docs/editors) for how to set up your editor. If your editor does not support Prettier, you can instead [run Prettier with a file watcher](https://prettier.io/docs/watching-files).
